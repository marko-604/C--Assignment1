\doxysubsection{Detailed Description}
Main driver for the \doxylink{class_tower}{Tower} Defense game. 

This file handles the initialization of the game map, tower placement, and the main game loop. It reads user input for map configuration and tower placement, spawns critters via a wave generator, and processes critter movement and tower attacks during game ticks.

Demonstrates the functionality of the game, the way that the game works is that there is a global tick that occurs every tick\+\_\+interval seconds, critters will move based on the ticks and towers will fire on creatures based on critters being in range and the tick counter.

\item game\+Critters\+: A global vector containing pointers to the critters currently in the game.

Main function that runs the \doxylink{class_tower}{Tower} Defense game. 

The function performs the following steps\+:
\begin{DoxyItemize}
\item Prompts the user for the map dimensions.
\item Configures the map by reading a series of coordinates to create a path, then setting the entry and exit points.
\item Validates the configured map.
\item Allows the user to place towers until points run out.
\item Enters the main game loop where\+:
\begin{DoxyItemize}
\item \doxylink{class_critter}{Critter} waves are generated periodically.
\item Each tick, critters move and are checked for reaching the exit (affecting player health).
\item Towers check for critters in range and attack them.
\item The map is updated and displayed.
\end{DoxyItemize}
\end{DoxyItemize}

The game loop ends when the tick count exceeds a predefined limit or the player\textquotesingle{}s health reaches zero.

\begin{DoxyReturn}{Returns}
int 0 if the game ends normally, or 0 if the player dies. 
\end{DoxyReturn}
